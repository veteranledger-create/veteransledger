Publish Pipeline

Validate records, stage output files, then promote to the live archive. Use Rollback to restore the previous snapshot if something goes wrong.

Step 1
Validate

Check all published records for conformance issues before writing any files.

›
Step 2
Stage

Validate and write output JSON to the staging directory. Does not touch live files.

›
Step 3
Promote

Atomically swap staged files into the live archive. Public site reflects changes immediately.

Recovery
Rollback

Restore the previous promoted snapshot. Use only if the current live version has a problem.
